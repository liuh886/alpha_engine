# SeekingAlpha-Style Quant Rating Strategy (Weekly 30 Equal-Weight)

## Status
- **Design Approved**: 2026-02-03 (user confirmed rules + assumptions)
- **Implementation**: pending

## Goal
Create a robust, reproducible baseline strategy that mimics “Seeking Alpha Quant Rating” mechanics:
- Pick **30 stocks**, **equal weight**
- **Buy/Include** only when `Quant Rating == Strong Buy` for **N consecutive trading days** (N is internal)
- **Risk/Tradability** filters (first version: price/liquidity/has-trades)
- **Weekly rebalance** to reflect latest ratings and relative strength
- **Cash handling**: dividends/sale proceeds accumulate during the week; reinvest **only on rebalance day**

This strategy’s signals MUST be generated by our trained model (not external SeekingAlpha data).

## Decisions (Locked for v1)
### Universe
- Candidate pool is **US** tickers from `configs/watchlist.yaml` under `us:`.

### Model Output + Rating
- Model is “XGBoost-style” **5-class classifier** producing `P(StrongBuy/Buy/Hold/Sell/StrongSell)`.
- Labels are defined by **future 20-trading-day relative return vs benchmark** (default benchmark: `QQQ`), using **daily cross-sectional 5-quantile bins**.
- At inference time, **rating is assigned by cross-sectional quantile mapping** of `P(StrongBuy)` (StrongBuy = top 20%, StrongSell = bottom 20%, etc.). This avoids hard-coded probability thresholds.

### Signal persistence filter
- `StrongBuy` must hold for **N=3 consecutive trading days** (`strongbuy_streak >= 3`) before eligible for inclusion.
- N is an internal hyperparameter (not publicly disclosed).

### Rebalance cadence
- Rebalance happens **weekly** on the **last trading day of each week** (holiday-safe).
- **No trading intra-week** (strictly weekly turnover).

### Trade timing (anti look-ahead)
- Use **t-1** signal to trade on **t** rebalance day close:
  - signal window: t-1
  - execution price assumption: t close (or equivalent backtest deal_price=close)

### Tradability / risk filters (v1)
Only price/volume derived filters (no M&A / SEC filing delay in v1):
- **Price cap**: `close < 10000` USD
- **Liquidity**: 20D average dollar volume >= **$10M/day** (initial default for v1)
- **Daily trades**: 20D window has `volume > 0` (or no missing trading days)

### Portfolio construction
- **Target holdings**: up to 30 names, equal weight.
- On rebalance day:
  - compute `TARGET` = eligible StrongBuy (streak>=3) + filters, sorted by `P(StrongBuy)`, take top 30.
  - if `|TARGET| < 30`, **do not backfill** (hold residual in cash).
- **Sell rule**: on rebalance day, sell any holding not in `TARGET`.
- **Re-weight rule**: on rebalance day, adjust holdings in `TARGET` back to equal weights.
- **Cash**: dividends/sale proceeds stay as cash intra-week, invested only on rebalance day.

## Open Items (v2+)
- Event filters:
  - “No major M&A” (acquirer/target)
  - “No SEC reporting delays”
- Alternative backfill policies if `|TARGET| < 30` (currently “hold cash”).
- Sensitivity analysis automation:
  - `N ∈ {1, 3, 5}`
  - `min_liquidity ∈ {1M, 5M, 10M, 20M}`

## Implementation Plan (High-Level)
1. **Data + features**
   - Ensure `data/watchlist` exposes needed fields for dollar volume:
     - Prefer `$money` if present; else fallback to `$close * $volume`.
2. **Model training**
   - Add training pipeline for 5-class classifier on horizon=20 relative returns.
   - Persist artifacts (model, config snapshot, label definition metadata).
3. **Signal inference**
   - Produce daily `P(StrongBuy)` per instrument.
   - Convert to rating via cross-sectional quantile bins.
   - Compute `strongbuy_streak`.
4. **Backtest strategy**
   - Add a new Qlib strategy class (BaseSignalStrategy-based) that:
     - trades only on weekly last trading day
     - enforces `TARGET` construction and equal-weight rebalancing
     - enforces cash-only intra-week policy
5. **Evaluation**
   - Track alpha/IR/DD/turnover plus:
     - average cash % / weeks with <30 holdings
     - weekly turnover distribution

## Acceptance Criteria (v1)
- Strategy runs end-to-end in backtest without look-ahead.
- Weekly rebalance schedule is correct (handles holidays).
- Produces stable `TARGET` behavior (streak>=3 gates entries).
- Outputs a clear report (holdings, cash %, trades, key metrics) and can be visualized in dashboard.
